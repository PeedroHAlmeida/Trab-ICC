Utilização dos comandos Git/GitHub:

  • init:

    O comando git init é um comando único que você usa durante a configuração inicial de um novo repositório. A execução desse comando cria um novo subdiretório .git no diretório de trabalho atual.

    Veja o exemplo:

  • clone:

    O comando git clone clona um repositório em um diretório recém-criado, cria o monitoramento remoto dos ramos para cada ramo no repositório clonado, cria e verifica um ramo inicial que é bifurcado do ramo ativo atualmente do repositório clonado .

    Veja o exemplo:

  • branch:

    O comando git branch permite criar, listar, renomear e excluir ramificações.

    Veja o exemplo:

    Não comando mostrou as branchs que o usuário já fez, como ainda não tinhamos feito nenhuma branch ele nos mostrou nada

  • checkout:

    O comando git checkout permite navegar entre ramificações criadas pelo git branch

    Veja o exemplo:

  • status:

    O comando git status exibe as condições do diretório de trabalho e da área de staging. Ele permite que você veja quais alterações foram despreparadas, quais não foram e quais arquivos não estão sendo monitorados pelo Git.

    Veja o exemplo:

  • diff:

    O comando git diff exibe as alterações entre a árvore de trabalho, o índice ou uma árvore, as alterações entre o índice e uma árvore, as alterações entre as duas árvores, nas alterações resultantes de uma mesclagem, nas alterações entre dois objetos gota ou nas alterações entre dois arquivos no disco.

    Veja o exemplo:

  • add:

    O comando git add em geral ele adiciona o conteúdo atual dos caminhos existentes como um todo, mas com algumas opções ele também pode ser utilizado para adicionar o conteúdo com apenas a parte das alterações aplicadas nos arquivos da árvore de trabalho ou remover os caminhos que não existam mais na árvore de trabalho.

    Veja o exemplo:

  • commit:

    O comando git commit cria um novo commit que tenha todos os conteúdos atuais do índice e a mensagem informada no registro log descrevendo as alterações.

    Veja o exemplo:

  • push:

    O comando git push permite que você envie (ou em tradução literal, empurre) os commits de sua branch e repositório Git local para o seu repositório remoto.

  • pull:

    O comando git pull é usado para buscar e baixar conteúdo de repositórios remotos e fazer a atualização imediata ao repositório local para que os conteúdos sejam iguais.

    Veja o exemplo:

  • revert:

    O comando git revert é usado para desfazer alterações ao histórico de commits do repositório.

  • merge:

    O comando git merge incorpora as alterações dos commits citados (desde o momento em que os seus históricos divergirem do ramo atual) para dentro do ramo atual.

  • stash:

    O comando git stash quando quiser gravar a condição atual do diretório ativo e do índice, porém queira voltar para um diretório funcional limpo.

  • rm:

    O comando git rm gera uma linha (na forma de um comando rm ) para cada arquivo removido.
