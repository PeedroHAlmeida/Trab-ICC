Utilização dos comandos Git/GitHub:

  • init:

    O comando git init é um comando único que você usa durante a configuração inicial de um novo repositório. A execução desse comando cria um novo subdiretório .git no diretório de trabalho atual.

    Veja o exemplo:

  • clone:

    O comando git clone clona um repositório em um diretório recém-criado, cria o monitoramento remoto dos ramos para cada ramo no repositório clonado, cria e verifica um ramo inicial que é bifurcado do ramo ativo atualmente do repositório clonado .

    Veja o exemplo:

  • branch:

    O comando git branch é utilizado para gerenciar branches (ramificações) em um repositório Git.

    O comando exibiu a branch de criação pois foi a unica feita até o momento

  • checkout:

    O comando git checkout permite navegar entre ramificações criadas pelo 'git branch'

    Veja o exemplo:

  • status:

    O comando git status é usado para exibir o estado atual do repositório de trabalho e da área de staging. Ele mostra quais alterações foram feitas, quais arquivos estão no staging e quais não estão, além de informar sobre qualquer commit pendente.

  • diff:

    O comando git diff é uma ferramenta poderosa para revisar e entender mudanças em seu código, é usado para visualizar as diferenças entre commits, branches, e o estado do diretório de trabalho. Ele é especialmente útil para detectar erros antes de fazer um commit e para revisar mudanças entre diferentes versões de seu código.

    Veja o exemplo:

  • add:

    Quando você modifica, adiciona ou exclui arquivos no diretório de trabalho, essas mudanças não são automaticamente incluídas no próximo commit. Você deve usar git add para indicar ao Git quais mudanças devem ser incluídas no próximo commit. O comando git add não faz um commit, ele apenas adiciona as mudanças ao índice (staging area).

  • commit:

    Quando você executa git commit, o Git pega todas as mudanças que foram adicionadas ao índice (usando git add) e as grava em um novo commit. Cada commit tem um identificador único e pode incluir uma mensagem descritiva, que ajuda a entender o que foi mudado e por quê.

    O comando git push é usado para enviar commits do repositório local para um repositório remoto. Ele transfere as mudanças que foram commitadas localmente para o repositório remoto, como o GitHub, GitLab, Bitbucket, etc. Aqui está uma explicação detalhada de como usar git push e exemplos de seu uso.

    O comando git pull é usado para buscar e baixar conteúdo de repositórios remotos e fazer a atualização imediata ao repositório local para que os conteúdos sejam iguais.

    O git revert cria um novo commit que aplica o inverso das mudanças feitas em um commit específico. Isso é útil para desfazer mudanças indesejadas enquanto mantém o histórico de commits intacto.
